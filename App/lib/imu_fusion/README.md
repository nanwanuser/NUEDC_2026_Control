# IMU 算法层

本目录是 ICM42688P 与官方 Fusion 库之间的适配层，负责传感器读取、低通滤波、零偏补偿和姿态输出。

## 文件说明

- imu_fusion.c/.h：保持项目原有 `IMU_Fusion*` 接口，内部调用 `App/lib/fusion` 中的官方 Fusion AHRS 和 Bias 算法。

## 职责边界

算法层应该负责：

- 消费来自驱动层的 IMU 数据。
- 维护四元数、欧拉角和陀螺仪零偏状态。
- 集中管理融合参数，方便后续调参。

算法层不应该负责：

- 直接访问 UART、GPIO、OLED 等外设。
- 控制主循环调度。
- 依赖 demo 专用传输或显示逻辑。

## 依赖方向

- 当前实现依赖：`icm42688p.h` 的传感器读取接口，以及 `Fusion.h` 的官方算法接口。
- 被：服务层调用。
- 不应该依赖：服务层和应用显示层。

## 调参入口

默认融合参数位于 `imu_fusion.c` 中的 `IMU_FusionGetDefaultConfig()`。当前按 ICM42688P 的 1 kHz、500 dps 配置设置，使用 NWU 坐标约定和无磁力计更新模式。

无磁力计时横滚角和俯仰角可由重力长期约束，航向角只能依靠陀螺仪积分，会随时间漂移。
