# IMU 算法层

本目录是 IMU 的算法层，负责对传感器数据进行处理和姿态估计，并尽量保持平台无关。

## 文件说明

- imu_fusion.c/.h：陀螺仪零偏估计、加速度低通滤波、Mahony 四元数融合、欧拉角输出。

## 职责边界

算法层应该负责：

- 消费来自驱动层的 IMU 数据。
- 维护四元数、欧拉角和陀螺仪零偏状态。
- 集中管理融合参数，方便后续调参。

算法层不应该负责：

- 直接访问 UART、GPIO、OLED 等外设。
- 控制主循环调度。
- 依赖 demo 专用传输或显示逻辑。

## 依赖方向

- 当前实现依赖：icm42688p.h 中的传感器数据类型和读取接口。
- 被：服务层调用。
- 不应该依赖：服务层和应用显示层。

## 调参入口

默认融合参数位于 imu_fusion.c 中的 IMU_FusionGetDefaultConfig()。后续要优化动态响应、静态稳定性和零漂，优先从这里开始。
