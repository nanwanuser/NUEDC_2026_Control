# IMU 驱动层

本目录是 IMU 的驱动层，只负责和 ICM42688P 硬件本体交互，不负责姿态解算、JustFloat 发送或 demo 调度。

## 文件说明

- icm42688p.c/.h：ICM42688P 的 SPI 驱动、寄存器初始化、原始数据与缩放数据读取、DRDY 标志处理。
- imu_board.h：本项目的板级宏定义，集中放置 CS 和 INT1 的引脚映射。
- IMU.md：整个 IMU 软件在本项目中的集成说明。

## 职责边界

驱动层应该负责：

- 调用 HAL 的 SPI、GPIO、DMA 和 Tick 接口。
- 完成 ICM42688P 初始化和寄存器访问。
- 向上层提供原始或物理量单位的 IMU 数据。
- 维护数据就绪中断标志。

驱动层不应该负责：

- 四元数解算。
- 串口遥测发送。
- OLED、控制逻辑或 demo 业务。

## 依赖方向

- 可以依赖：Core/Inc、HAL、CMSIS。
- 可以被：算法层、服务层调用。
- 不应该依赖：服务层。

## 维护建议

如果以后 CubeMX 再次生成代码，把 IMU 板级宏单独留在 imu_board.h，可以避免被 main.h 覆盖。
